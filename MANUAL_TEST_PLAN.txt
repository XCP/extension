WALLET MANUAL TEST PLAN
========================

Prerequisites:
- 3 addresses in wallet (Address A, B, C)
- Small amount of BTC on Address A (0.001 BTC / 100,000 sats should be plenty)
- Some XCP on Address A if testing named asset creation (0.5 XCP per named asset)

Legend:
[ ] = Not tested
[x] = Passed
[!] = Issue found (add notes)


PHASE 1: OFF-CHAIN (NO FEES)
============================

1. [x] Sign Message
   Route: /actions/sign-message
   Steps: Enter "test message" > Sign > Copy signature
   Notes:

2. [x] Verify Message
   Route: /actions/verify-message
   Steps: Paste address, message, signature from step 1 > Verify
   Expected: Shows valid signature
   Notes:

[!] Note: Both of these worked nicely, however, the submit button is outside of the white container that holds the inputs. I believe it's more common for us to put the submit button inside the white container.
